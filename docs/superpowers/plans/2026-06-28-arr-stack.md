# Arr Stack Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Revive the archived *arr stack and deploy it as a local-only media automation suite alongside the existing Jellyfin instance.

**Architecture:** Five containers (qBittorrent, Prowlarr, Sonarr, Radarr, Lidarr) in a single docker-compose on an internal bridge network. All web UIs bound to localhost only — no Traefik exposure. Jellyfin stays untouched in its own compose. Volume paths reuse existing env vars for media directories plus one new env var for downloads.

**Tech Stack:** Docker Compose, linuxserver.io images

## Global Constraints

- PUID/PGID: 33:33 (matches existing Jellyfin)
- No Traefik labels — localhost-only access
- No Jellyfin in this compose — already in `docker/services/jellyfin/`
- No Readarr — dropped per design
- Reuse existing env vars: `SHOWS_DIR`, `MOVIES_DIR`, `MUSIC_DIR`
- New env var: `QBITTORRENT_DOWNLOAD_DIR`

### Task 2: Rewrite docker-compose.yml to match design

**Files:**
- Rewrite: `docker/services/arr-stack/docker-compose.yml`

- [ ] **Step 1: Replace the compose file entirely**

Write the following to `docker/services/arr-stack/docker-compose.yml`:

```yaml
services:
  qbittorrent:
    image: lscr.io/linuxserver/qbittorrent:latest
    container_name: qbittorrent
    environment:
      PUID: 33
      PGID: 33
      TZ: "${TZ}"
      WEBUI_PORT: 8080
      TORRENTING_PORT: 6881
    volumes:
      - ${DOCKERDIR}/arr/qbittorrent:/config
      - ${QBITTORRENT_DOWNLOAD_DIR}:/data/downloads
    ports:
      - "127.0.0.1:8080:8080"
      - "127.0.0.1:6881:6881"
      - "127.0.0.1:6881:6881/udp"
    restart: unless-stopped
    networks:
      - arr-network

  prowlarr:
    image: lscr.io/linuxserver/prowlarr:latest
    container_name: prowlarr
    environment:
      PUID: 33
      PGID: 33
      TZ: "${TZ}"
    volumes:
      - ${DOCKERDIR}/arr/prowlarr:/config
    ports:
      - "127.0.0.1:9696:9696"
    restart: unless-stopped
    networks:
      - arr-network

  sonarr:
    image: lscr.io/linuxserver/sonarr:latest
    container_name: sonarr
    environment:
      PUID: 33
      PGID: 33
      TZ: "${TZ}"
    volumes:
      - ${DOCKERDIR}/arr/sonarr:/config
      - ${SHOWS_DIR}:/data/tvshows
      - ${QBITTORRENT_DOWNLOAD_DIR}:/data/downloads
    ports:
      - "127.0.0.1:8989:8989"
    restart: unless-stopped
    networks:
      - arr-network

  radarr:
    image: lscr.io/linuxserver/radarr:latest
    container_name: radarr
    environment:
      PUID: 33
      PGID: 33
      TZ: "${TZ}"
    volumes:
      - ${DOCKERDIR}/arr/radarr:/config
      - ${MOVIES_DIR}:/data/movies
      - ${QBITTORRENT_DOWNLOAD_DIR}:/data/downloads
    ports:
      - "127.0.0.1:7878:7878"
    restart: unless-stopped
    networks:
      - arr-network

  lidarr:
    image: lscr.io/linuxserver/lidarr:latest
    container_name: lidarr
    environment:
      PUID: 33
      PGID: 33
      TZ: "${TZ}"
    volumes:
      - ${DOCKERDIR}/arr/lidarr:/config
      - ${MUSIC_DIR}:/data/music
      - ${QBITTORRENT_DOWNLOAD_DIR}:/data/downloads
    ports:
      - "127.0.0.1:8686:8686"
    restart: unless-stopped
    networks:
      - arr-network

networks:
  arr-network:
    name: arr-network
    driver: bridge
```
